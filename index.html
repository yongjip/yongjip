Building an AI-Powered Text-to-SQL System to Scale Data Access

Ad-hoc data requests were a bottleneck. Simple questions could be answered quickly, but more complex requests often required hours of research to identify the right tables, columns, join patterns, and business definitions.

Instead of trying to make humans write SQL faster, I focused on making the underlying knowledge reusable and accessible to AI.

flowchart TD A[User Question] --> B[Domain / Table Selection] B --> C[Structured Knowledge Layer] C --> C1[Table Definitions] C --> C2[Column Semantics] C --> C3[Join Patterns] C --> C4[Business Terms] C --> D[AI Agent (Codex / Claude / Gemini)] D --> E[SQL Query Generation] E --> F[Run Query in Athena] F --> G{Syntax / Execution OK?} G -- No --> D G -- Yes --> H[Result QA / Data Validation] H --> I{Quality Check Passed?} I -- No --> D I -- Yes --> J[Deliver to Human] J --> K[Use Result / Update Knowledge if Needed] K --> C

The key is not just the model. It is how the problem is structured.

  • Built a structured metadata layer with table definitions, column meanings, join patterns, and business terms
  • Reduced ad-hoc query time from 4–10 hours to under 1 hour
  • Increased monthly ad-hoc capacity from around 20 to 50+
